Payment flow
01
Request
POST to the endpoint with your token address. Receive a 402 response with the USDC payment address and exact amount in the X-Payment-Required header.
02
Pay
Send USDC on Solana mainnet to the address in the header. Keep your transaction signature — that's your payment proof. No accounts, no sessions, no subscriptions.
03
Receive
Re-POST the same request with X-Payment: <base64 sig> header. Signature is verified on-chain. Service runs. Structured JSON returned.
Example — sol_quick_scan
bash 
# Step 1 — get payment terms
curl -X POST https://api.solprobe.xyz/api/x402/quick_scan \
  -H "Content-Type: application/json" \
  -d '{"token_address":"EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v"}'

# ← 402 Payment Required
# X-Payment-Required: {"amount":"0.02","token":"USDC","network":"solana",
#                      "address":"8Fk...SolProbeWallet","memo":"solprobe:quick_scan"}

# Step 2 — re-send with payment proof
curl -X POST https://api.solprobe.xyz/api/x402/quick_scan \
  -H "Content-Type: application/json" \
  -H 'X-Payment: eyJ0eF9zaWduYXR1cmUiOiI1eEs5Li4uIiwibmV0d29yayI6InNvbGFuYSJ9' \
  -d '{"token_address":"EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v"}'

# ← 200 OK
